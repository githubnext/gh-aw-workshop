# Interactive Workshop: Getting Started with GitHub Agentic Workflows

## Workshop Goals
- Understand what agentic workflows are and when to use them.
- Help developers prototype workflow automation quickly.
- Help program managers align workflow outcomes with delivery goals.

## Track 1: Developers
1. Identify one repetitive engineering task suitable for automation.
2. Define clear inputs, outputs, and guardrails for an agentic workflow.
3. Run the workflow and capture learnings for iteration.

## Track 2: Program Managers
1. Define a workshop objective tied to a measurable outcome.
2. Map stakeholders, review checkpoints, and decision points.
3. Validate workflow output quality and readiness criteria.

## Starter Structure for a Basic Agentic Workflow
- **Context:** What problem are we solving?
- **Task:** What should the agent do daily?
- **Constraints:** What must the agent avoid?
- **Output:** What artifact should be generated?

## Recommended Starting Point
Start with **Palettes** and the **/slash-agentic workflow designer** skill to quickly scaffold and refine your first workflow.
Open Palettes from Copilot Chat as your guided prompt workspace, then run the slash-agentic workflow designer skill there to generate an initial workflow draft you can iterate on.
If you are new to these tools, start with the GitHub Copilot documentation: https://docs.github.com/en/copilot
Daily Focus rotates on a UTC day boundary.

## Daily Reflection Prompts
- What worked well today?
- What should be changed in tomorrow's run?
- What additional guidance would improve output quality?
